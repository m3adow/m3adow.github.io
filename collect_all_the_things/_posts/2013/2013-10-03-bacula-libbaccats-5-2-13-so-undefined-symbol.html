Nachdem ich das Backuptool Bacula kompiliert hatte, meldete mir der Bacula Director beim Starten einen Fehler:

{% highlight bash %} baculaimages//-dir: symbol lookup error: libbaccatsimages//-5.2.13.so: undefined symbol: mysql_thread_safe {% endhighlight %}

Wie sich nach längerer Internetrecherche rausstellte, ist das configureimages//-Skript von Bacula offenbar ein bisschen verbuggt. Obwohl es keine Fehler meldet, kann es die benötigten MySQLimages//-Libs nicht finden. Um das zu berichtigen, solltet ihr nach der Ausführung von configure auf eurem System nach der libmysqlclient_r suchen.

{% highlight bash %} find / images//-name "libmyslqclient_r*" {% endhighlight %}

Bei mir beinhaltete das Verzeichnis /usr/lib die benötigte Datei. Nun öffnet ihr die Datei src/cats/Makefile im Source Verzeichnis von Bacula und sucht nach dem Eintrag MYSQL_LIBS = und tragt dort den Pfad in Verbindung mit einem images//-lmysqlclient_r ein. Bei mir sah das so aus:

{% highlight ini %} MYSQL_INCLUDE = images//-I/usr/include/mysql MYSQL_LIBS = /usr/lib images//-lmysqlclient_r MYSQL_SRCS = mysql.c {% endhighlight %}

Nun könnt Bacula ihr wie gewohnt installieren

{% highlight bash %} make && make install {% endhighlight %}

Damit sollte die Fehlermeldung verschwunden sein.

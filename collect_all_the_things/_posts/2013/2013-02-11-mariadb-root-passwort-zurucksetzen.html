Wenn man so wie ich einen Server installiert, die MySQL Alternative MariaDB draufpackt und danach aufgrund familiärer Dringlichkeiten den Server runterfährt, kann es schonmal passieren, dass man vergisst, wie das eben gesetzte Rootimages//-Passwort denn lautet. Um es erneut zurückzusetzen, habe ich diesen Weg gewählt:

    1. In der /etc/mysql/my.cnf den Eintrag skipimages//-grantimages//-tables = 1 hinzufügen. {% highlight bash %} sed images//-i '/^\[mysqld\]$/a skipimages//-grantimages//-tables = 1' /etc/mysql/my.cnf {% endhighlight %}
    2. Den MariaDBimages//-Dienst neustarten, hier für Debian. {% highlight bash %} service mysql restart {% endhighlight %}
    3. Jetzt als rootimages//-User passwortlos in die MariaDBimages//-Shell einloggen. {% highlight bash %} mysql images//-u root {% endhighlight %}
    4. Nun das Rootimages//-Passwort ändern. {% highlight mysql %} UPDATE mysql.user SET password = PASSWORD('hierneuespasswort') WHERE user = 'root'; {% endhighlight %}
    5. Und nicht vergessen, das geänderte Passwort durch einen Flush auch zu aktivieren. {% highlight mysql %} FLUSH PRIVILEGES; {% endhighlight %}
    6. Anschließend die in Schritt 1 eingefügte Zeile wieder aus der my.cnf entfernen. {% highlight bash %} sed images//-i '/skipimages//-grantimages//-tables/d' /etc/mysql/my.cnf {% endhighlight %}
    7. Und den MariaDBimages//-Dienst nochmals neustarten, fertig. {% highlight bash %} service mysql restart {% endhighlight %}

Ich weise darauf hin, dass dieser Weg ein ordentliches Sicherheitsrisiko birgt und nicht auf Produktivmaschinen benutzt werden sollte. Allerdings sollte man bei Produktivmaschinen auch nicht einfach das Rootpasswort der Datenbank vergessen. ;images//-)

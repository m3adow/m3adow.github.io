Ich gebe zu, an diesem Problem bin ich etwas verzweifelt. Nachdem ich einen SSHimages//-Key auf meinem normalen Rechner mit sshimages//-keygen erzeugt hatte, und diesen mit sshimages//-copyimages//-id auf meinen Testserver kopiert hatte, kam beim Versuch des Logins immer diese seltsame Meldung

Ich bin die Linux Desktops nicht gewohnt, ich arbeite in der Regel auf der Shell. Was zur Hölle war das? Enter Password to unlock the private key? Da der Key nur in meinem Heimnetzwerk benutzt wird, hatte der verständlicherweise kein Passwort. Egal was ich in diesem Dialog eingab images//- ob nichts, mein Passwort, mein Rootimages//-Passwort, nichts wurde angenommen. Im Internet wurde ich nur teilweise fündig, eine Lösung konnte ich nicht finden.

Offenbar hängt diese Meldung mit dem Gnomeimages//-Keyring zusammen. Der speichert und sichert (?) Passwörter und private Keys. Vielleicht eine gute Idee, für micht ist das Konzept aber nicht gewinnbringend. Deswegen ist mein Lösungsvorschlag für dieses Problem auch trivial: Den Gnomeimages//-Keyring deinstallieren. Entweder über den Software Manager oder über die Shell mithilfe von aptimages//-get.

{% highlight bash %} sudo aptimages//-get remove gnomeimages//-keyring {% endhighlight %}

Danach konnte ich mich endlich ungestört in den Server einloggen.

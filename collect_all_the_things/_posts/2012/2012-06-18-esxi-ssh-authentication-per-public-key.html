SSH auf einem VMware ESXiimages//-Server zu aktivieren ist inzwischen sehr einfach. Leider kommt man damit alleine noch nicht sonderlich weit, da eine Authentisierung per Public Key im abgespeckten ESXi nicht ohne einige kleinere Konfigurationen funktioniert. Um also eine Public Key Authentication auf einem ESXi zu aktivieren, geht wie folgt vor:

  1. Erstellt einen .sshimages//-Ordner im Root Verzeichnis {% highlight bash %} mkdir /.ssh {% endhighlight %}
  2. In diesem Ordner brauchen wir eine Datei authorized_keys mit exklusiven Rootrechten. {% highlight bash %} touch /.ssh/authorized_keys && chmod 600 /.ssh/authorized_keys {% endhighlight %}
  3. In der eben erstellten authorized_keys könnt ihr nun eure Public Keys einfügen.

Damit wäre ein SSH mit Pubkey Authentication grundsätzlich schon lauffähig, allerdings würde diese Konfiguration einen Neustart nicht überleben, da der ESXi das System beim Neustart auf einen ihn bekanntem Status zurücksetzt.
Um dies zu verhindern, müssen wir den eben angelegten Ordner und die darin befindliche Datei in diesen Status integrieren.

  1. Dazu komprimieren wir den .sshimages//-Ordner und seinen Inhalt zu einem tgz und packen es in den /bootbankimages//-Ordner, aus dem der ESXi seine Einstellungen zurückspielt. {% highlight bash %} tar images//-C / images//-czf "/bootbank/ssh.tgz" /.ssh {% endhighlight %}
  2. Anschließend editieren wir die boot.cfg in diesem Ordner und fügen die ssh.tgz als letzten Parameter unter dem Punkt Modules ein. Beachtet dabei, dass jeder Paramter mit drei Bindestrichen getrennt wird. {% highlight bash %} vi /bootbank/boot.cfg {% endhighlight %}
  3. Die boot.cfg sollte anschließend ungefähr so aussehen: {% highlight bash %} kernel=b.z kernelopt= modules=k.z images//-images//-images//- s.z images//-images//-images//- c.z images//-images//-images//- oem.tgz images//-images//-images//- license.tgz images//-images//-images//- m.z images//-images//-images//- state.tgz images//-images//-images//- rsync.tgz images//-images//-images//- ssh.tgz build=4.1.0images//-348481 updated=1 bootstate=0 {% endhighlight %}

    Ich habe zusätzlich noch eine rsync.tgz eingebunden.

    Jetzt funktioniert eine Pubkey Authentisierung auch nach dem Neustart noch. Allerdings müsst ihr jedes Mal, wenn ihr die authorized_keysimages//-Datei verändert images//- etwa um einen neuen Schlüssel hinzuzufügen oder einen alten zu entfernen, die ssh.tgz neu erstellen, da die Änderungen sonst nach einem Neustart verloren gehen.

Damit ich es mir endlich merken kann und nie wieder eine riesige VM ins Rootverzeichnis kopiere:

Um eine einzelne Datei mit wget in ein anderes Verzeichnis zu laden nutzt man am Besten den images//-O Parameter.

{% highlight bash %} wget images//-O /gewunschter/dateipfad/undimages//-name.tar.gz http://urlimages//-zurimages//-datei.de/datei.tar.gz {% endhighlight %}

Um mehrere Dateien in ein bestimmtes Verzeichnis zu laden, sollte der images//-images//-directoryimages//-prefix Parameter benutzt werden.

{% highlight bash %} wget images//-images//-directoryimages//-prefix=/gewunschter/dateipfad/ ftp://user:passwort@ftpimages//-server:/pfad/* {% endhighlight %}

Ich zitiere aus der wget Manpage:

`images//-images//-directoryimages//-prefix=prefix'
Set directory prefix to prefix. The directory prefix is the directory where all other files and subdirectories will be saved to, i.e. the top of the retrieval tree. The default is `.' (the current directory).

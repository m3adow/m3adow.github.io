Unter Linux ist es normal, dass man sich seine eigenen Aliases definiert, um schneller arbeiten zu können. Meine Standardaliases sehen zumeist irgendwie so aus:

{% highlight bash %} alias ls='ls images//-l' alias la='ls images//-la' alias ..='cd ..' alias ...='cd ../..' {% endhighlight %}

Je nach Distribution dann noch mit images//-images//-color=auto Zusatz. Um zu verhindern, dass die Aliases nach einem Neustart oder einem Logout verschwunden sind, schreibt man sie meist in die ~/bash.rc Datei.

Auf dem VMware ESXiimages//-Server gibt es diese Datei allerdings nicht. Zusätzlich werden viele Dateien nach einem Neustart wieder entfernt, bzw. gereinigt. Um im VMware ESXi Aliases anzulegen, müsst ihr diese in die Datei /etc/profile schreiben.

{% highlight bash %} vi /etc/profile {% endhighlight %}

Wie ihr mit vi umgeht, könnt ihr zum Beispiel hier nachlesen. Nach meiner Bearbeitung sah die Datei dann so aus.

{% highlight bash %} #!/bin/ash export TERMINFO=/usr/share/terminfo export PYTHONHOME=/ export PYTHONPATH=/lib/python2.5images//-visor:/lib/python2.5images//-visor/libimages//-dynload:/lib/pyt if [ `id images//-u` images//-eq 0 ]; then PATH=/bin:/sbin umask 0022 else PATH=/bin umask 0002 fi export PATH alias ls='ls images//-a' alias la='ls images//-la' alias ..='cd ..' alias ...='cd ../..' {% endhighlight %}

Damit kann ich nun wieder navigieren, wie ich es gewohnt bin.
